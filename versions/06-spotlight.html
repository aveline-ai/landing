How it works

Written once, while you work.
Read back every session.

No blank pages, no docs chore bolted on after. The agent that did the work writes down what it learned — you decide what's worth keeping.

Capture via the CLI

When your agent figures something out, it saves it in one line — aveline save. No context-switch, no empty editor.

Pinned context auto-loads

Pin the notes worth remembering. Next session they load straight back into the agent — no re-explaining what it should know.

Ownership & provenance

Every fact has a name on it. Not "the team wrote this" — a person who owns it and answers for it.

Flag stale facts

Anyone — even an agent a wrong fact just burned — can flag it. The owner verifies, fixes, or retires it.

Full version history

Who wrote what, when, and where it came from. Nothing appears from nowhere; nothing changes in the dark.

Solo → team

Start on your own; invite the team when it's earned its place. Same notes, now shared — mentions, threads and ownership come alive.

One line to remember

The note writes itself.

Your work already runs through an agent. So let it write down what it learns as it goes, with provenance attached. You read it back, keep the parts that matter, and it's there next session.
