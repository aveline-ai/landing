01 The problem

Every wiki you've had quietly went stale.

It always starts tidy. Then the work moves on, and nobody goes back to fix the page. Six months later the docs are lying to you — so you stop trusting them, so you stop writing them. Same quiet death, every time.

deploy-runbook.md edited 9mo ago
Staging deploys run through Jenkins.
Ping Priya for the staging creds.
Nightly cron fires at 2am UTC.
Jenkins is gone. Priya left in March.
02 What changed

Your work already runs through an agent.

So stop hand-writing the docs. The agent that did the work writes down what it learned, as it goes. You read it. You keep the parts that matter. The notes stay alive because keeping them alive is finally part of the work.

~/work
› aveline save --pin --title "staging deploy" saved · owned by you · pinned to workspace memory ›
03 How it works
01 Capture

It writes as it goes

When your agent figures something out, it saves it with one line — aveline save. No blank page, no context-switch, no chore bolted on after.

02 Curate

You keep what matters

Pin the notes worth remembering. Pinned notes are your real memory; everything else is a searchable scratchpad.

03 Recall

It starts already knowing

Next session, the pinned notes load straight back into your agent. No re-explaining what it should already know.

04 Why you'll trust it

A wiki is only worth reading if you believe it. So every fact carries the things that make it believable — the raw spec:

Mechanic Guarantee What it means
owner Every fact has a name on it Someone owns it and answers for it. Not "the team wrote this" — a person.
history Every change is on the record Who wrote what, when, and where it came from. Nothing appears from nowhere; nothing changes in the dark.
flag Stale facts get called out Anyone — even an agent a wrong fact just burned — can flag it. The owner verifies, fixes, or retires it.
version Pinned to a known-good state aveline marks the version everyone — and every agent — reads by default. Drafts stay out of the way.
05 Starts with you
SOLO → TEAM

Begin solo — it's the memory your agent has been missing between sessions. When it has earned its place, invite your team into the same workspace. Same notes, now shared. Mentions, threads and ownership come alive. Nothing about how it works changes.
