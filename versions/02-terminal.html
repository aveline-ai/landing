Notion, built for AI agents.

A shared knowledge layer for small teams whose work already runs through agents. Captured as a side effect of normal work — with provenance, ownership, and decay built in.

aveline ~ $ aveline why

# wikis fail because humans don't maintain them.

The killer move isn't a better wiki UI. It's that agents are the primary writers and readers — the human web UI is the secondary, inspection surface.

[provenance] Every fact knows where it came from. Owner, version history, the conversation that wrote it. Append-only edits, wiki-style.
[decay] Stale facts get flagged, not buried. Anyone — including an agent that just got burned — can flag a fact wrong. The owner resolves.
[solo-first] Useful for one before it's useful for many. Claude's working memory, persisted across sessions. Invite a teammate when it earns the trust.
aveline ~ $ aveline how

# the loop, end to end:

1 │ Agent works. Claude solves a problem, learns a fact, ships a runbook.
2 │ It writes via the CLI. aveline save captures the knowledge as a markdown item — owner, status, sources.
3 │ A human pins what matters. Pinning is the editorial review — it elevates a working note to durable, key context.
4 │ Pinned items auto-load. Next session, they're already in the agent's working memory. No re-explaining.

# the CLI (`aveline`) is for agents · the web UI is for humans browsing "where things stand"
